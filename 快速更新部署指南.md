## 🎯 上传到GitHub（网页版）

### 方法1：网页上传（推荐，最简单）

#### 步骤1：打开GitHub仓库
访问你的GitHub仓库：`https://github.com/你的用户名/你的仓库名`

#### 步骤2：删除旧文件（如果存在）
1. 点击 `vercel_app.py` 文件
2. 点击右上角的「...」→「Delete file」
3. 填写commit信息：`update: 删除旧版本代码`

对 `vercel_requirements.txt` 重复上述步骤。

#### 步骤3：上传新文件

**上传 `vercel_app.py`：**
1. 在仓库首页，点击「Add file」→「Upload files」
2. 拖拽或选择 `vercel_app.py` 文件
3. 填写commit信息：`feat: 支持企业内部机器人，实现自动回复功能`
4. 点击「Commit changes」

**上传 `vercel_requirements.txt`：**
1. 再次点击「Add file」→「Upload files」
2. 拖拽或选择 `vercel_requirements.txt` 文件
3. 填写commit信息：`chore: 添加加密库依赖`
4. 点击「Commit changes」

**上传 `企业内部机器人配置指南.md`：**
1. 再次点击「Add file」→「Upload files」
2. 拖拽或选择 `企业内部机器人配置指南.md` 文件

## 📋 配置环境变量

### 1. 在钉钉开放平台创建企业内部机器人
按照《企业内部机器人配置指南.md》中的步骤操作。

### 2. 获取配置信息
- AppKey
- AppSecret
- Token
- EncodingAESKey

### 3. 在Vercel中配置环境变量
1. 打开Vercel项目
2. 点击「Settings」→「Environment Variables」
3. 添加以下4个变量：

| 名称 | 值 |
|------|-----|
| `DINGTALK_APP_KEY` | 你的AppKey |
| `DINGTALK_APP_SECRET` | 你的AppSecret |
| `DINGTALK_TOKEN` | 你的Token |
| `DINGTACK_AES_KEY` | 你的EncodingAESKey |

4. 点击「Save」

### 4. 重新部署
1. 返回项目首页
2. 点击「Redeploy」
3. 等待部署完成

### 5. 再次验证
访问：`https://你的项目名.vercel.app/`

现在 `config` 中的所有项都应该显示 `✓`：

```json
{
  "config": {
    "APP_KEY": "✓",
    "APP_SECRET": "✓",
    "TOKEN": "✓",
    "AES_KEY": "✓"
  }
}
```

## 🔧 故障排查

### 问题1：部署失败
**检查**：
- 文件是否正确上传
- `vercel_requirements.txt` 是否存在

### 问题2：服务运行但配置全部是 ✗
**原因**：环境变量未配置
**解决**：按照上面的步骤配置环境变量

### 问题3：配置是 ✓ 但机器人不回复
**检查**：
1. 钉钉开放平台的消息接收地址是否正确
2. 是否勾选了「接收群消息」事件
3. 机器人是否已发布

### 问题4：签名验证失败
**原因**：Token或AES Key配置错误
**解决**：
- 确保Vercel中的环境变量与钉钉开放平台的配置完全一致
- 检查是否有多余的空格
